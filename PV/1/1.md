# Adresování a správa paměti - Garbage collecting, Reference/ukazatele, Struktura paměti programu

## Adresování
Místo celých hodnot se přeposílá ukazatel na paměť, kde je hodnota uložena.

- každý byte v paměti RAM má vlastní adresu
- Virtuální paměť
- Paměťové stránky (Memory Page)
    - mmap / VirtualAlloc syscalls

- pointer
- stack / heap

## správa paměti

- garbage collector
    
    - Kód, který kotroluje, které objekty již nejsou využívané a ty smaže.
    - Python, Java, C#

- Resource Aquisition Is Initialization
    
    - Vloží destruktor na konec kódu kde je objekt využíván
    - Vložení objektu do funkce, jiného objektu = nová inicializace
    - C++

- borrow checking
    
    - Striktní pravida
    - pouze jeden vlastník, který objekt smaže
    - Rust     

- manualní
    
    - Programátor musí sám zavolat destruktor obkejtu
    
    - C, Zig

- reference counting

    - Každý objekt má číslo referencí, které ho využívají
    - Když má 0 referencí objekt se smaže

## Garbage collecting

- Pomalé
- Programátor nemusí přemýšlet nad pamětí
- Stromová struktura referencí

GC skenuje stack a hledá objekty, na které už není žádný ukazatel na stacku.
GC smaže nevyužívané objekty.

**Java** - 3 vrstvy životnosti objektů

## Reference / ukazatele

**ukazatel**
- Proměnná, která obsahuje adresu do paměti RAM.
- Ukazuje kde v paměťi je jiná proměnná.

**reference**
- Odkazuje na existující proměnnou.
- Syntactic sugar nad ukazately
- Reference na objekt zaručuje, že objekt existuje a není NULL


## Struktura paměti programu

- stack
- heap

**stack**
- Zásobník FIFO
- na linuxu 4MB
- Rychĺý
- Obsahuje proměnné funkcí, return adresu funkce
- Rekurze může udělat stack overflow

**heap**
- Halda (Stromová struktura, nebo linked list)
- Pomalá
- Obsahujé objekty, které jsou moc velké na stack
- Obsahujé objekty, které jijichž velikost není známá při kompilaci (user input)
